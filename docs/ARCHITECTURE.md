# Architecture Principles

## Philosophy

Keep the project simple.

Optimise for readability over cleverness.

---

## Components

Create reusable components whenever possible.

Avoid duplicated UI.

---

## Pages

Each page should have a single responsibility.

---

## AI

AI requests should always pass through the server.

Never expose API keys.

---

## Data

Prefer caching before requesting AI.

---

## Code Quality

Small components.

Clear naming.

Explain major structural changes before implementing.

---

## Project Structure

Never create another Next.js project.

Never duplicate folders.

Always modify the existing application.
